Getting Started with MCP Memory: Add Persistent Memory to Claude Code, Cursor & More

Give your AI coding assistant the ability to remember everything—preferences, project context, past decisions. Set up in under 5 minutes.

The Model Context Protocol (MCP) is revolutionizing how AI tools work together. Instead of isolated assistants that forget everything between sessions, MCP enables persistent AI memory that follows you across tools and conversations.

In this guide, you'll learn how to add an MCP memory server to your favorite AI coding tools. Whether you're using Claude Code, Cursor, Windsurf, or any other MCP-compatible editor, you'll have persistent memory working in minutes.

What You'll Need

  • An MCP-compatible tool — Claude Code, Cursor, Windsurf, Continue, etc.
  • Node.js 18+ or Docker (depending on setup method)
  • 5 minutes of your time

Option 1: Claude Code (Fastest)

If you're using Claude Code, adding persistent memory is a single command:

claude mcp add remembra -- npx -y remembra-mcp

That's it. Claude Code will now remember your preferences, project context, and past decisions across all sessions.

What Happens Behind the Scenes

When you run this command:

  1. Claude Code registers the Remembra MCP server
  2. The server starts automatically when Claude Code launches
  3. Memories are stored locally in ~/.remembra/
  4. Relevant context is retrieved automatically for each conversation

Your AI now has AI agent memory that persists forever.

Option 2: Cursor

Cursor supports MCP through its settings. Here's how to add persistent memory:

Step 1: Open MCP Settings

Go to Cursor Settings → Features → MCP Servers

Step 2: Add the Memory Server

Click "Add Server" and enter:

{
  "name": "remembra",
  "command": "npx",
  "args": ["-y", "remembra-mcp"]
}

Step 3: Restart Cursor

Restart the editor to activate the MCP connection.

Now Cursor will maintain persistent context across all your coding sessions.

Option 3: Any MCP-Compatible Tool

Most MCP-compatible tools follow a similar pattern. Look for MCP or "context servers" in settings, then add:

Command: npx
Args: -y remembra-mcp

Or if your tool uses a config file:

{
  "mcpServers": {
    "remembra": {
      "command": "npx",
      "args": ["-y", "remembra-mcp"]
    }
  }
}

Option 4: Self-Hosted with Docker

Want complete control over your AI memory? Run your own server:

docker run -d -p 8787:8787 \
  -v remembra-data:/data \
  remembra/remembra

Then point your MCP config to your local server:

{
  "name": "remembra",
  "command": "npx",
  "args": ["-y", "remembra-mcp"],
  "env": {
    "REMEMBRA_URL": "http://localhost:8787"
  }
}

Your data stays on your machine, under your control. See our complete self-hosting guide for more details.

Using Your Memory-Enabled AI

Once set up, your AI assistant gains several new abilities:

Automatic Learning

Tell your AI your preferences once:

"I always use TypeScript with strict mode, Tailwind for styling, and prefer functional components."

It remembers forever. No more repeating yourself.

Context Recall

Ask questions that reference past work:

"What was that authentication approach we discussed last week?"

The memory server retrieves relevant context automatically.

Entity Tracking

Your AI understands who and what you're talking about:

"What does Sarah think about the new API design?"

It connects "Sarah" to past discussions about team members and their opinions.

Verifying It Works

Test your setup with a simple interaction:

  1. Tell your AI something specific: "Remember that I'm building a SaaS app called Acme"
  2. Close the conversation completely
  3. Start a new session and ask: "What app am I building?"

If the AI remembers "Acme," your persistent AI context is working perfectly.

Tips for Better Memory

  • Be explicit about preferences — "Always use X" or "Never do Y" creates strong memories
  • Name things consistently — Call your project the same name every time
  • Correct mistakes — If the AI gets something wrong, tell it; the correction is remembered
  • Review occasionally — Ask "What do you remember about this project?" to verify context

Troubleshooting

Memory not persisting?

  • Ensure the MCP server is listed as "connected" in your tool's status
  • Check that ~/.remembra/ directory exists and is writable
  • Try restarting your editor

Using Docker and getting connection errors?

  • Verify the container is running: docker ps | grep remembra
  • Test the API: curl http://localhost:8787/health
  • Check logs: docker logs remembra

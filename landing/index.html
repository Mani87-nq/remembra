A memory layer you can operate, not just call.

Remembra is built for agent systems that need continuity, accountability, source-aware recall, and context that improves over time.

Conversation Ingest

Turn full conversations into durable facts, entities, and deduped memories with optional dry runs and streaming progress.

  • Fact extraction + importance scoring
  • Entity extraction and speaker attribution
  • SSE progress for larger sessions

Temporal Truth

Memories that understand time: TTL, natural decay, as-of relationship queries, and stale-context cleanup.

  • TTL presets (session → permanent)
  • Ebbinghaus-based decay
  • as_of historical queries

Recall Pipeline

Not just vectors. Combine semantic search, BM25, graph retrieval, reranking, and token-aware context shaping.

  • Vector + BM25 fusion
  • Graph-aware recall
  • Token-aware context optimization

Security Controls

Practical production controls for memory systems: encryption support, PII handling, scoped keys, rate limits, sanitization, and audit trails.

  • AES-256-GCM encryption at rest
  • PII detection & redaction
  • API key auth + rate limiting
  • Full audit logging

Conflict Review

  • Project-scoped conflict lists
  • Review stats for operators

API advantage

Memory APIs for the parts agents usually lose.

The next level for Remembra is not another generic vector endpoint. It is the operational API surface around memory: ingestion, conflicts, inbox handoffs, meetings, observability, plugins, and exports that help teams trust what agents remember.

remembra-api
POST /api/v1/ingest/conversation/stream # extract facts with progress
POST /api/v1/debug/recall # explain why memory returned
GET /api/v1/conflicts/stats # review contradictory facts
POST /api/v1/inbox/send # target another agent
POST /api/v1/meetings/summarize # convert calls into action memory
ingest

Conversation memory, not chat logs

Extracts facts, entities, speaker context, deduped memories, and processing stats from whole conversations.

debug

Explainable recall

Shows semantic, keyword, graph, recency, entity, access, rerank, and final scores for operators.

conflicts

Contradiction queue

Gives teams a place to inspect and resolve memory conflicts instead of letting old facts quietly win.

inbox

Agent-to-agent pickup

Lets agents leave targeted tasks, context, status, and acknowledgements for the next tool in the chain.

meetings

Meeting continuity

Builds pre-meeting briefs and post-meeting summaries from calendar, transcript, and memory context.

webhooks

Evented memory systems

Dispatch memory lifecycle events to downstream tools with signing, retries, plugins, and audit visibility.

Up and Running in 5 Minutes

From zero to AI memory in three steps

1

Deploy

One command to run Remembra with Docker:

docker run -d -p 8787:8787 remembra/remembra
2

Connect All Agents

One command configures supported AI tools:

pip install remembra
remembra-install --all --url http://localhost:8787

Detects Claude, Codex, Cursor, Windsurf, Gemini, and more where supported.

3

Use

Store and recall memories:

# Store
memory.store("User likes dark mode")

# Recall
memory.recall("preferences")
